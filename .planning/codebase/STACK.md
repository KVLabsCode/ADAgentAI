## Configuration

**Environment:**
Frontend `.env`:
- `NEXT_PUBLIC_API_URL` - Backend API URL (default: http://localhost:3001)
- `NEXT_PUBLIC_AGENT_URL` - Chat agent service URL (default: http://localhost:5001)
- `NEXT_PUBLIC_POSTHOG_KEY` - Product analytics API key
- `NEXT_PUBLIC_SANITY_*` - Sanity CMS configuration
- `NEXT_PUBLIC_NEON_AUTH_URL` - Neon Auth endpoint for client-side operations

Backend `.env` (shared by API and chat agent):
- `DATABASE_URL` - Neon PostgreSQL connection string (required)
- `ANTHROPIC_API_KEY` - Claude LLM API key (required for chat)
- `OPENROUTER_API_KEY` - Alternative LLM provider
- `VOYAGE_API_KEY` - Embeddings provider
- `NEON_AUTH_URL` - Neon Auth JWT validation endpoint
- `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` - Google OAuth credentials
- `POLAR_ACCESS_TOKEN` - Billing platform API token
- `LANGSMITH_API_KEY` - LLM debugging/observability
- `SENTRY_DSN` - Error tracking
- `POSTHOG_API_KEY` - Analytics tracking
- `SMTP_USER`, `SMTP_PASS` - Email service (Gmail SMTP)
- `INTERNAL_API_KEY` - Service-to-service authentication
- `TOKEN_ENCRYPTION_SECRET` - Encrypts stored OAuth tokens

**Build:**
- `tsconfig.json` - TypeScript compiler options (strict mode, path aliases)
- `next.config.ts` - Next.js configuration (React Compiler, image optimization, experimental features)
- `playwright.config.ts` - E2E test configuration
- `tokens/style-dictionary.config.mjs` - Design token build settings

**Database:**
- `backend/api/src/db/schema.ts` - Drizzle ORM schema definitions
- Migrations auto-generated by Drizzle Kit (`drizzle-kit push`)
